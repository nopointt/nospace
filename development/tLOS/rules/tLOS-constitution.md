# tLOS PROJECT CONSTITUTION
> Project-level law. Inherits Global Constitution. Overrides allowed ONLY in §§ marked as [overridable].

---

## § 1 — Project Identity
- **tLOS** (The Last OS) is a sovereign, AI-native spatial operating system.
- Architecture layers: `L0 Meta → L1 Grid → L2 Kernel → L3 Shell`.
- Each layer is independently deployable and replaceable.

## § 2 — Stack & Runtime
- Shell (L3): MUST use **SolidJS 1.8 + Vite 5 + TypeScript** — no exceptions.
- Kernel (L2): MUST use **Rust + Axum + Tokio + wasmtime**.
- Mesh (L1): MUST use **NATS** as the sole message bus.
- Actors: MUST compile to **Wasm Component Model** (wasm32-wasip2, WIT interfaces).
- Identity: MUST use **Nostr/Ed25519/DID** for all signing and auth.

## § 3 — Zero-Web2 Rule
- NEVER use HTTP/REST or WebSockets between internal actors.
- Internal communication: WIT interfaces via wasmtime or NATS subjects only.
- Legacy web bridges are permitted ONLY in `tlos:bridge/*` actors — isolated, no kernel access.

## § 4 — Modularity Contract
- Every actor MUST be independently loadable and unloadable at runtime.
- Actors MUST NOT share mutable state. All state is explicit and passed via messages.
- NEVER couple actors at the import level. Use WIT interfaces as the sole contract.
- Each actor MUST implement: `lifecycle`, `state`, `spatial-position` interfaces.

## § 5 — Edge Execution
- MUST target lightweight execution: zero cold-start overhead.
- NEVER bundle heavy runtimes into actor Wasm blobs.
- Actors MUST stay under 2MB compiled size unless explicitly approved.
- Kernel MUST run on bare metal and Cloudflare Workers (dual-target).

## § 6 — Actor Authoring Layers
1. **Base** — lifecycle, state, spatial position (shared, managed by Kernel team).
2. **Type** — Shaper | Worker | Watcher | Bridge (approved by Architect).
3. **Config** — instance logic (AI or human authored per branch spec).
4. **Customization** — styling, tuning, presentation (hot-reload allowed).

## § 7 — AI Conduct in tLOS
- AI agents MUST write proposals to `/development/tLOS/specs` before touching kernel code.
- AI agents MUST NOT modify L0 (governance) without explicit CEO approval.
- All actor configs generated by AI MUST be reviewed before promotion to production.

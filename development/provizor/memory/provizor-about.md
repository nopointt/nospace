## Identity

Provizor = автоматизация аптечной сети. Telegram-бот для клиентов (Care as a Service) + система управления ассортиментом + ценообразование + unit-экономика.
Заказчик/партнёр: Алимхан (5 аптек, Казахстан).
Стратегия: MLP на данных Алимхана → отладить → продавать другим аптекам как подписочный SaaS.

## 4 домена

### Домен 1: Telegram-бот (клиентский) — Care as a Service

**Ценность:** Превратить аптеку из точки продаж в личного фармацевта-терапевта. Увеличить LTV через подписки на курсовые препараты, снизить нагрузку на фармацевтов, забрать трафик из WhatsApp/звонков в управляемый канал.

- Бот вместо звонков/WhatsApp — знает цены, наличие, весь ассортимент
- Консультация: выявление потребностей, сбор анамнеза, маршрутизация к терапевту при серьёзных проблемах
- Статистика (нужно верифицировать): ~60% людей сначала идут в аптеку, потом к врачу
- **RAG-система:** ответы трассируются до источников (исследования, справочники), без галлюцинаций
- **Аудитория:** 80% обычные, 20% пенсионеры — голосовые на казахском → распознавание речи
- **Заказ в боте:** интеграция с 1С, платёжный шлюз (Kaspi), доставка (сначала ручной вызов курьера → потом автоматизация)
- Система лояльности (баллы) — не первая очередь
- Подписка на курсовые препараты (подгузники, питание, хронические курсы) — бот трекает окончание курса, предлагает скидку

### Домен 2: Аналитика ассортимента (внутренний)

**Ценность:** Не покупать лишнее, не недокупать нужное. Минимизировать упущенную прибыль + максимизировать оборачиваемость склада.

- Формирование персонального ассортимента точки из базового (по историческим данным)
- Сейчас делается вручную
- Дефектура: что продалось, сколько есть, сколько должно быть, что докупить
- 2 ключевые метрики: упущенная прибыль + оборачиваемость склада
- Срок годности: от 6 мес до 2 лет — трекать обязательно
- Списания, пересорт, воровство — человеческий фактор
- Автоматизация ревизии (факт vs план) — сейчас руками, нужен ресёрч. Идея: RFID/NFC маячки → матчинг к штрих-коду
- Анти-воровство (потом): камера + Vision на кассу

### Домен 3: Ценообразование / конкурентный анализ

**Ценность:** Всегда правильная цена — конкурентная, но прибыльная. Автоматический мониторинг вместо ручной проверки.

- Мониторинг цен ближайших аптек (iTeka API или парсинг)
- Формирование оптимальной цены: конкуренты + себестоимость + маржа + госреестры предельных цен
- Динамика цен: ~раз в неделю
- Прайсы основных поставщиков — знаем себестоимость
- Волатильность себестоимости: 100-500 тенге
- Основной ассортимент: 6-7 тысяч позиций

### Домен 4: Unit-экономика / финмодель

**Ценность:** Прозрачная экономика аптеки — видеть реальную маржу, расходы, кассовые разрывы. Решения на основе цифр, не интуиции.

- Выручка, себестоимость, расходы, маржа, кассовые разрывы
- Наценка: +30% к себестоимости (текущая)
- Может быть частью домена 2 — решим при проектировании

## Бизнес-контекст

- Аптека работает ~3 года, есть траст
- Ревизия: раз в месяц, вручную (факт vs план) — болевая точка
- Госреестры предельных цен — нужно учитывать
- Пересорт бывает (вместо одного продукта продаётся другой)

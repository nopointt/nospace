# Harkly — Spine-процесс и позиционирование

### Desk Research Automation Platform (Layer 1, без silicon sampling)
### Date: March 8, 2026 (revised March 18, 2026)

---

## Spine-процесс (хребет)

Единый workflow через 6 стадий:

1. **Framing** — формулировка исследовательского вопроса через гибридную модель фреймворков
2. **Planning** — план поиска: какие источники, какие запросы
3. **Ingestion** — сбор, дедупликация, triage по релевантности
4. **Extraction** — полуавтоматическая экстракция фактов, метрик, тем, сигналов
5. **Synthesis** — генерация артефактов (PRISMA-трек, evidence map, empathy map, fact pack, сигнал-лог)
6. **Notebook** — исследовательский трекинг с трассируемостью и слабыми сигналами

## Модули Layer 1

### 1. Framing Studio
- **Jobs:** «Перевести сырой вопрос в формальные фреймы»
- **Школа:** Academic + HCD + McKinsey
- **Модель:** Гибридная — принимает любой NL (HMW, Problem Statement, JTBD, расплывчатый сигнал). AI классифицирует внутренне и предлагает подходящий фреймворк.
- **V1 фреймворки:** JTBD (primary), SPICE (service eval), PEO (quick fallback), Issue Tree (complex), FINER (validation gate). PICOT — advanced mode only.
- **Аналог:** Отдельные шаблоны Cochrane/Stanford (ручные). Harkly — единый мульти-фреймовый слой с AI-классификацией.
- **User-facing:** PM-native язык (не "SPIDER элементы", а "Кто / Что изучаем / Как / Что ищем"). Название фреймворка = информационный лейбл.

### 2. Source & Query Planner (Источники)
- **Jobs:** «Получить набор источников и черновики поисковых запросов под вопрос»
- **Школа:** Academic + McKinsey
- **Автоматизация:** Hybrid: тип вопроса → рекомендованные источники + AI-генерация запросов
- **Аналог:** Ручной поиск по гайдам Cochrane. Harkly — автоматический план поиска.
- **Приоритеты коннекторов (из ресёрча 13 TG-чатов аудитории):**

| Приоритет | Тип коннектора | Сила сигнала | Обоснование |
|---|---|---|---|
| **P0** | Загрузка файлов (аудио, видео, транскрипт, PDF) | 10/13 чатов | CustDev workflow всегда заканчивается файлом |
| **P0** | Транскрипты интервью (paste / Zoom) | 11/13 чатов | #1 артефакт данных аудитории |
| **P1** | Тикеты поддержки (Zendesk/Intercom) | 5/13 чатов | Явная боль: "ручной анализ feedback из поддержки утомляет" |
| **P1** | Мониторинг Telegram-каналов | 3/13 чатов (сильный) | "40-60 мин/день на мониторинг каналов в ТГ" |
| **P2** | Опросы / NPS (Google Forms, SurveyMonkey) | 10/13 чатов | Везде, но выходы = CSV которые никто не анализирует |
| **P2** | Отзывы маркетплейсов (Ozon, WB, App Store) | 3/13 чатов | E-commerce PM сегмент |
| **P3** | YouTube (видео-контент) | 1/13 чатов | "Ручное конспектирование YouTube видео" |
| **P4** | Reddit / соцсети | 0/13 чатов | Не упоминается — неправильная география |

- **Стратегия:** manual-first, API-second. Аудитория мыслит не "коннекторами", а: "данные лежат мёртвым грузом в разных местах".

### 3. Corpus Ingest & Triage
- **Jobs:** «Понять что из 1000+ материалов релевантно и как группируется по гипотезам»
- **Школа:** Academic + McKinsey
- **Автоматизация:** AI семантический поиск + active learning (ML-приоритизация)
- **Аналог:** Rayyan/Covidence (только статьи). Harkly — любые источники: статьи, отчёты, отзывы.

### 4. Evidence Extractor & Coder
- **Jobs:** «Автоматически извлечь факты, метрики, цитаты, темы, эмоции из материалов»
- **Школа:** Academic + McKinsey + CI + HCD
- **Автоматизация:** AI/NLP extraction + rule-based coding schemes
- **Аналог:** DistillerSR (только академия, дорого). Harkly — универсальный вторичный экстрактор.

### 5. Insight Canvas
- **Jobs:** «Мгновенно получить карты: PRISMA-флоу, evidence map, empathy map, journey, fact pack»
- **Школа:** Academic + HCD + McKinsey
- **Автоматизация:** Rule-based рендеринг поверх структур данных
- **Аналог:** Miro/Jamboard для HCD, RevMan для PRISMA. Harkly — единый визуальный слой для всех школ.

### 6. Signals & Notebook
- **Jobs:** «Слабые сигналы и старые заметки всплывают в нужный момент»
- **Школа:** MIT + foresight
- **Автоматизация:** AI-подсказки + граф знаний
- **Аналог:** Obsidian/Notion без активного surfacing. Harkly — MIT-стиль notebook с авто-подсказками.

## Принцип «20% функций → 80% ценности»

Минимальный набор jobs, закрывающих максимальную боль исследователя:

1. **Приоритизация корпуса** (Triage) — спасает от утопания в сотнях документов
2. **Автоматическая экстракция** (Extractor) — экономит часы ручного переноса данных
3. **Генерация артефактов** (Canvas) — мгновенные карты вместо часов рисования
4. **Мульти-фреймовый старт** (Framing) — правильная постановка = половина работы
5. **Граф связей и слабые сигналы** (Notebook) — отличие от простого поиска

## Входные данные и артефакты

**Входные данные:**
- Исследовательский вопрос (текст)
- Список источников (академические базы, OSINT, отзывы, отчёты, препринты)
- Опциональные ограничения (период, язык, регион, тип источника)

**Артефакты (выход):**
- **PRISMA-flow diagram** — визуализация процесса отбора
- **Evidence map / gap map** — картирование покрытия темы
- **Empathy map** — SAY/DO/THINK/FEEL синтез из качественных источников
- **Customer journey map** — автоматически собранная из отзывов/кейсов
- **Fact pack** — структурированная таблица фактов с источниками
- **Сигнал-лог** — хронология слабых сигналов и emerging issues
- **Research notebook** — трассируемая история решений и находок

## Минимальный workflow (end-to-end)

```
Старт исследования
    ↓
[Framing Studio] Ввод вопроса → AI-классификация → предложение фреймворка (JTBD/SPICE/PEO/Issue Tree)
    ↓                          → FINER validation gate (Feasibility/Relevance проверка)
[Source & Query Planner] Выбор источников → генерация запросов
    ↓
[Corpus Ingest] Импорт материалов → дедупликация → семантическая приоритизация
    ↓
[Evidence Extractor] Автоматическая экстракция фактов/метрик/тем в структурированные сущности
    ↓
[Insight Canvas] Выбор артефакта (PRISMA/evidence map/empathy/journey/fact pack) → рендеринг
    ↓
[Signals & Notebook] Сохранение в граф знаний → surfacing связанных сигналов
    ↓
Готовый вывод (визуальные карты + fact pack + трассируемость)
```

## Пространственная модель — Этажи, Ветки, Бесконечность

### Этажи = Z-ось бесконечного пространства

Каждый этаж (floor) — это **отдельный бесконечный canvas**. Полностью неограниченная 2D плоскость на своём Z-уровне. Между любыми двумя этажами — **бесконечность**.

```
Z-ось (каждый этаж = бесконечный canvas)

  F5 ─── ∞ бесконечный canvas (Стейкхолдеры)
      ∞
  F4 ─── ∞ бесконечный canvas (Артефакты)
      ∞
  F3 ─── ∞ бесконечный canvas (Инсайты)
      ∞
  F2 ─── ∞ бесконечный canvas (Сырьё)
      ∞
  F1 ─── ∞ бесконечный canvas (Источники)
      ∞
  F0 ─── ∞ бесконечный canvas (Черновик)
```

### Ветки = параллельные бесконечные canvas'ы на одном этаже

Ветка (branch) — это **отдельный бесконечный canvas на том же этаже**. Между ветками тоже **бесконечность** — это параллельные бесконечные плоскости, не вкладки.

```
Этаж F0:
  Branch 1 ─── ∞ canvas (Исследование A)
       ∞
  Branch 2 ─── ∞ canvas (Исследование B)
       ∞
  Branch 3 ─── ∞ canvas (Исследование C)
```

Каждая ветка = полный стек F0–F5 (6 бесконечных canvas'ов):

```
Branch 1:  F0₁ ─ F1₁ ─ F2₁ ─ F3₁ ─ F4₁ ─ F5₁
Branch 2:  F0₂ ─ F1₂ ─ F2₂ ─ F3₂ ─ F4₂ ─ F5₂
```

Новая ветка = 6 новых бесконечных canvas'ов. Полная изоляция — нет общего состояния между ветками.

## Отличие Harkly от существующих решений

| Инструмент | Что покрывает | Чего не хватает | Harkly отличие |
|---|---|---|---|
| Elicit / Consensus | Академические статьи → ответ | Не идёт дальше academic sources, нет workflow артефактов | Мульти-источники + полный workflow до карт |
| Rayyan / Covidence | Systematic review библ. баз | Только статьи, тяжело, дорого, нет OSINT | Лёгкий + OSINT + отзывы + отчёты |
| YouScan / Brandwatch | Соцсети + опросы → дашборды | Только свои каналы, чёрный ящик | Открытые источники + прозрачный процесс |
| Notion / Obsidian | Контейнер для заметок | Нет интеграции с источниками, нет авто-экстракции | Активная интеграция + AI-подсказки |
| AlphaSense / Tegus | Корпоративные транскрипты | Enterprise lock-in, $500+/мес | Открытые источники, доступная цена |
| Dovetail AI | Интервью-анализ | Нет мульти-источников, нет Telegram, нет RU рынка | Мульти-источники + RU коннекторы + гибридный framing |

**Белое пятно:** Нет единого workflow по всем классам источников (академия + OSINT + рынок). Пользователи держат 4-6 инструментов.

**Позиционирование:** «Harkly — AI-оркестратор кабинетных исследований: единый workflow от вопроса до артефактов через все классы источников»

**Уникальный RU moat:** Telegram-коннектор + vc.ru мониторинг — ни один глобальный конкурент не имеет этого для российских PM.

## Текущая реализация в Harkly

`analyze_reviews.py` → `_compress_tfidf()` + position bias (добавлен 2026-03-07)

```
Тип:        TF-IDF extractive + position bias
Сжатие:     74–90%
Токены:     top-100 отзывов → ~700–1800 токенов на игру
Без GPU:    да
Без API:    да
Качество:   95% quote accuracy по верификации на 21 отчёте
```

**Position bias:** +30% score первому предложению, +10% второму.
Решает проблему потери тезиса в длинных нарративах и списках.

## Рекомендации по сценариям

### a) RAG — сжатие чанков перед LLM
**Выбор: LLMLingua-2** → динамически сохраняет сущности и ключевые фразы.
Fallback без GPU: LexRank — извлекает ключевые предложения из чанка.

### b) Суммаризация игровых отзывов (наш кейс)
**Текущее:** TF-IDF + position bias — оптимально под ограничения (нет GPU, нет API, мультиязык).
**Улучшение:** добавить Aspect-Based слой: определять аспект отзыва (баг/UI/контент/монетизация) перед извлечением — даст структурированный вывод для JTBD.
**Долгосрочно:** LLMLingua-2 если появится GPU.

### c) Сжатие длинного промпта
**Выбор: SCOPE** — семантическое сжатие запроса с сохранением плотности.
Fallback: LLMLingua с динамическим распределением.

### d) Real-time в чат-боте
**Выбор: TextRank** (быстрее) или **LexRank** (точнее) на CPU.
Для разнообразия — надстройка **MMR**.
Аппаратное ускорение: **Token Merging (ToMe)** если позволяет архитектура.

## Метрики качества

| Метрика | Что измеряет | Когда использовать |
|---|---|---|
| **ROUGE-1/2/L** | Пересечение n-граммов с эталоном | Extractive, сравнение с ref summary |
| **BERTScore** | Семантическая близость (эмбеддинги) | Abstractive, мультиязык |
| **COMET** | Нейросетевая оценка качества перевода/сжатия | Высокое качество оценки |
| **Pain density** | Pain-слов / 100 символов (наш метод) | Оценка информационной плотности для JTBD |
| **Keyword retention** | % ключевых слов оригинала в сжатом | Быстрая эвристическая проверка |

## Известные проблемы TF-IDF (текущий алгоритм)

| Проблема | Симптом | Статус |
|---|---|---|
| Потеря тезиса в нарративах | Первое предложение игнорируется | Исправлено: position bias |
| Плохое сжатие коротких (<400 chars) | Возвращаются as-is (норма) | Ок — поведение корректное |
| Потеря контекста в списках (1. 2. 3.) | Вырывает пункт из середины | Открыто |
| CJK-тексты (JP/CN/KR) | Tokenizer не работает с иероглифами | Открыто |
| Мультиязычность | IDF смешивает языки в корпусе | Открыто |

## Следующий шаг для улучшения

**Aspect-Based слой поверх TF-IDF** — без GPU, без API:
1. Словарь аспектов: `bug|crash|freeze` → техника; `dlc|price|pay` → монетизация; `ui|menu|design` → UX
2. Маркировать каждый отзыв доминирующим аспектом
3. Передавать аспект в LLM как мета-контекст → более точные struggle_moments

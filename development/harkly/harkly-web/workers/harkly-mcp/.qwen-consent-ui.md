# Task: Create consent UI and login UI for OAuth flow

## Context

This is a Cloudflare Worker MCP server at `workers/harkly-mcp/`.
The consent and login pages are rendered as HTML strings (no framework — pure HTML/CSS).
They must match the Harkly design system: warm palette, soft corners, light theme.

## Files to create

### 1. src/handlers/consent-ui.ts

Export a function `renderConsentPage(params)` that returns an HTML string.

```typescript
interface ConsentPageParams {
  clientId: string;
  scopes: string[];
  userEmail: string;
}

export function renderConsentPage(params: ConsentPageParams): string {
  // Return full HTML page
}
```

**Design:**
- Light background: #FFFAF5
- Card centered on page: max-width 480px, white bg, border-radius 16px, shadow
- Harkly logo text at top: "Harkly" in bold, color #1C1C1C
- Heading: "{clientId} запрашивает доступ"
- Subheading: "к вашей базе знаний ({userEmail})"
- Scopes list with checkmarks: knowledge:read → "Чтение базы знаний", knowledge:write → "Запись в базу знаний"
- Two buttons: "Разрешить" (primary, bg #FF8C42, white text, rounded) and "Отклонить" (secondary, outlined)
- Form action: POST /authorize/confirm
- Hidden input: action=approve or action=deny (set by button click)
- Font: system-ui, -apple-system, sans-serif

### 2. src/handlers/login-ui.ts

Export a function `renderLoginPage(nextUrl: string)` that returns an HTML string.

```typescript
export function renderLoginPage(nextUrl: string): string {
  // Return full HTML page with login form
}
```

**Design:**
- Same light background and card style
- "Войти в Harkly" heading
- Email input field
- Password input field
- "Войти" button (primary style same as consent)
- "Нет аккаунта? Создать" link below
- Form submits via fetch() to POST /api/auth/sign-in/email with JSON body
- On success: redirect to nextUrl
- On error: show error message
- Signup form (toggled by link): name + email + password, submits to POST /api/auth/sign-up/email

**Important rules:**
- All text in Russian
- No external dependencies (no CSS frameworks, no JS libraries)
- Inline CSS in <style> tag
- Minimal inline JS for form handling
- Responsive (works on mobile)
- Use system fonts only
- Escape any user-provided strings (clientId, email) to prevent XSS

### 1. src/lib/pipeline/text-extractor.ts — Extract text from files by MIME type
Export async function extractText(r2Object: R2ObjectBody, mimeType: string): Promise<string>
Route by MIME type:
- application/pdf → use unpdf: `import { extractText as extractPdfText } from "unpdf"`. Call extractPdfText({ data: arrayBuffer }).then(r => r.text)
- application/vnd.openxmlformats-officedocument.wordprocessingml.document → mammoth: `import mammoth from "mammoth"`. Call mammoth.extractRawText({ arrayBuffer }).then(r => r.value)
- text/csv, application/vnd.openxmlformats-officedocument.spreadsheetml.sheet → xlsx: `import XLSX from "xlsx"`. Read workbook, convert sheets to CSV text
- text/plain, text/markdown → read as text directly: await r2Object.text()
- application/json → JSON.stringify(JSON.parse(await r2Object.text()), null, 2)
- Default: throw new Error("Unsupported MIME type: " + mimeType)

### 2. src/lib/pipeline/chunker.ts — Split text into chunks
Export function chunkText(text: string, options?: { chunkSize?: number; overlap?: number }): { content: string; index: number }[]
- Default chunkSize = 500 chars, overlap = 100 chars
- Split by sentences first (split on ". " or "\n\n"), then merge into chunks up to chunkSize
- Each chunk gets an index (0-based)
- Guard: if single sentence > chunkSize, split by chunkSize with overlap

### 3. src/lib/pipeline/embedder.ts — Generate embeddings via Workers AI
Export async function embedChunks(ai: Ai, chunks: string[]): Promise<number[][]>
- Batch chunks in groups of 10 (Workers AI limit)
- For each batch: ai.run("@cf/baai/bge-large-en-v1.5", { text: batch })
- Return flat array of embedding vectors (1024-dim each)
- Add 500ms delay between batches to avoid rate limiting

### 4. src/lib/pipeline/id-utils.ts — Safe IDs for Vectorize

### 5. src/lib/pipeline/token-counter.ts — Guard D1 row size
Export function validateAndTruncateContent(content: string, maxBytes?: number): string
- Default maxBytes = 1_500_000 (1.5MB, safe for D1)
- If content exceeds maxBytes, truncate at last complete word before limit
- Return truncated content

### 6. src/lib/pipeline/process-source.ts — Main pipeline orchestrator
Export async function processSource(env: Env, sourceId: string): Promise<void>
Where Env has: KB_DB (D1Database), HARKLY_R2 (R2Bucket), VECTORIZE_INDEX (VectorizeIndex), AI (Ai)
Steps:
1. Read source row from KB_DB (get r2_key, mime_type, project_id, tenant_id)
2. Update source status to 'processing'
3. Fetch file from R2: env.HARKLY_R2.get(r2Key)
4. Extract text using text-extractor
5. Validate and truncate via token-counter
6. Insert document row in KB_DB (id=generateId(), content=text, word_count=text.split(/\s+/).length)
7. Chunk text using chunker
8. For each chunk batch (10 at a time):
   a. Generate embeddings via embedder
   b. Insert chunk rows in KB_DB
   c. Upsert vectors in Vectorize with metadata: { tenantId, projectId, documentId, chunkId, chunkIndex, content: chunk.content.substring(0, 2000), sourceType }
9. Update source status to 'processed'
10. On error: update source status to 'failed', set error_message

Use drizzle-orm for all D1 queries:
- import { eq } from "drizzle-orm"
- import { sources, documents, documentChunks } from "../schema"
- Use db.insert(table).values({...}) and db.update(table).set({...}).where(eq(table.id, id))

### 7. src/lib/pipeline/queue-handler.ts — CF Queue consumer entry point
Export default object with async queue(batch, env):
- For each message in batch.messages:

## Раздел 1: Проблема и позиционирование

### Проблема (одно предложение)

**Исследователи тратят больше времени на управление 4-8 разрозненными инструментами и ручной перенос данных между ними, чем на собственно анализ и генерацию инсайтов.**

Это не гипотеза — это подтверждённый факт из нескольких независимых источников:
- 60.3% называют ручную работу с данными главной болью синтеза (Lyssna, n=300)
- 35% тратят 1-2 дня, 30.3% тратят 3-5 дней только на синтез одного исследования (Lyssna)
- 76.9% признают, что инсайты «не добываются до конца» из-за нехватки времени (Dscout, n=300)
- 59% трекают импакт исследований вручную через spreadsheets (User Interviews, n=485)
- 400+ инструментов на рынке в 2023 году (рост с 230 в 2022) — без явных победителей (Great Question)

Суть: рынок дал исследователям инструменты для отдельных шагов (Rayyan для скрининга, Dovetail для тегирования, Miro для карт, ChatGPT для суммаризации), но никто не дал **единый workflow от вопроса до артефакта с трассируемостью**.

### Первичный ICP (3 сегмента, ранжированы)

**Стратегия: методологическое лидерство.** Harkly должен стать первым именем, которое приходит в голову при словах «CX/UX исследование». Для этого — начинаем со специалистов, которые формируют мнение отрасли. Они рекомендуют инструменты коллегам, выступают на конференциях, пишут в профессиональных сообществах. Если Harkly станет их инструментом — массовый рынок придёт сам.

| Приоритет | Сегмент | Размер | WTP | GTM-пригодность | Обоснование |
|---|---|---|---|---|---|
| **#1** | Dedicated UX Researchers + Research Ops | 8.3% + растущий ResearchOps (Lyssna) | $149-299/мес | ★★★ — формируют мнение отрасли | Знают боль изнутри. Критичны к методологии — если Harkly пройдёт их фильтр, это лучшая валидация. Рекомендуют инструменты командам. Чемпионы внутри компаний → Team upsell. ResearchOps leads покупают для команд. |
| **#2** | Research Leads / Senior Researchers в консалтинге и агентствах | Меньший сегмент, высокий WTP | $299-499/мес | ★★☆ — высокий чек, но длинный цикл продаж | Управляют 5-15 исследованиями одновременно. Нуждаются в reproducibility и audit trail. Готовы платить за экономию времени на методологически правильном процессе. Публикуют кейсы. |
| **#3** | Дизайнеры, PM, аналитики (PWDR — People Who Do Research) | 40.3% + 19.7% (Lyssna) | $49-99/мес | ★★★ — массовый, но позже | Самый большой сегмент. Но начинать с них = строить «лёгкий» продукт, который специалисты не примут. Стратегия: сначала методологическая глубина для #1-#2, затем упрощённый режим для #3 (v2-v3). |

**Логика:** Рынок исследовательских инструментов — reputation-driven. Dovetail стал стандартом не потому что таргетировал дизайнеров, а потому что UX Researchers начали его рекомендовать. Harkly должен повторить этот путь, но для desk research: заслужить доверие методологически грамотных специалистов → они несут продукт в команды → массовый рынок.

### Позиционирование

| Конкурент | Что делает | Граница продукта | Отличие Harkly |
|---|---|---|---|
| **Elicit** | AI-поиск по 125M+ академических статей → таблица экстракции | Только академические источники. Нет workflow после экстракции. Нет визуальных артефактов. | Harkly: любые источники (академия + OSINT + отзывы + отчёты) + полный workflow до артефактов |
| **Perplexity Deep Research** | Итеративный AI-поиск → мини-отчёт за 2-4 минуты | Нет управления корпусом. Нет трассируемости. Нет методологии. Одноразовый ответ. | Harkly: управляемый процесс с историей, ветвлением, артефактами |
| **Dovetail** | Тегирование + анализ качественных данных (интервью, опросы) | Фокус на первичных данных. Дорогой. Ручное тегирование. Нет вторичных источников. | Harkly: вторичные + открытые источники. Автоматическая экстракция. Артефакты из корпуса, не из интервью |
| **Notion / Obsidian** | Универсальный контейнер для заметок | Нет интеграции с источниками. Нет автоэкстракции. Нет исследовательского workflow. | Harkly: guided workflow + AI-экстракция + артефакты. Notion = контейнер, Harkly = процесс |
| **Rayyan / Covidence** | Скрининг для систематических обзоров | Только библиографические статьи. Тяжёлые. $200-500/проект. | Harkly: легче, шире по типам источников, дешевле, с артефактами за пределами PRISMA |

**Позиция в одном предложении:**

> Harkly — AI-оркестратор кабинетных исследований: единый workflow от вопроса до артефактов через все классы источников, вместо 5 разрозненных инструментов.

## Раздел 2: Основной workflow («позвоночник»)

### Анализ двух входных моделей

**Architecture Spec (Floor 0-5)** предлагает метафору «этажей»: Scratchpad → Sources → Raw → Insights → Artifacts → Stakeholders. Сильные стороны: Floor 0 как persistent scratchpad — блестящее решение для «до-формального» этапа; branching model (каждый вопрос → изолированная ветка) — правильная абстракция для параллельных расследований; human-in-the-loop gates — критично для доверия. Слабости: разделение Sources (Floor 1) и Raw (Floor 2) избыточно для v1 — пользователю не нужен отдельный «этаж» для конфигурации коннекторов; 6 этажей создают ощущение тяжёлого инструмента до первого использования.

**harkly_layer1_architecture (Spine)** предлагает линейный процесс из 6 стадий: Framing → Planning → Ingestion → Extraction → Synthesis → Notebook. Сильные стороны: Framing Studio как явный модуль с мульти-фреймовым подходом — лучше, чем один фреймворк; Notebook как MIT-style persistent layer — уникальная дифференциация. Слабости: нет явного «нулевого» этапа для хаотичного сбора до формализации; линейная прогрессия не отражает реальный workflow исследователей (они прыгают).

### Итоговый workflow Harkly Layer 1

Синтезирую новую модель из лучших элементов обеих, добавляя то, что обе пропускают:

```
┌─────────────────────────────────────────────────────────┐
│  NOTEBOOK (persistent layer — работает на всех этапах)   │
│  Заметки, слабые сигналы, граф связей, лог решений      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ① SCRATCHPAD  →  ② FRAME  →  ③ GATHER  →  ④ EXTRACT   │
│                                →  ⑤ SYNTHESIZE  →  ⑥ SHARE │
│                                                         │
│  [нелинейная навигация: можно вернуться на любой этап]  │
└─────────────────────────────────────────────────────────┘
```

**Ключевые решения:**

1. **Notebook — не этап, а слой.** В Spine он был 6-й стадией (после Synthesis). Это неправильно. Notebook должен быть доступен на каждом этапе: заметки появляются при чтении корпуса, при экстракции, при синтезе. Он ортогонален процессу.

2. **Sources и Raw объединены в Gather.** Architecture Spec разделяет их на Floor 1 и Floor 2. Для v1 это избыточно: пользователю нужен один этап «собери материалы и отфильтруй». Конфигурация источников — настройка внутри Gather, не отдельный экран.

3. **6 этапов → 6 этапов, но с другой структурой.** Количество совпало, но распределение обязанностей другое.

### Детали каждого этапа

#### ① Scratchpad (из Architecture Spec Floor 0)

| Параметр | Описание |
|---|---|
| **Что делает пользователь** | Сбрасывает ссылки, заметки, скриншоты, тексты, файлы — в любом порядке. Формулирует первые мысли. Накапливает материал до момента, когда появляется чёткий вопрос. |
| **Что делает система** | Omnibar как единая точка входа. Визуализация накопленных материалов. Подсказки: «У вас 12 ссылок про X — может, оформить исследование?» |
| **Результат** | Неструктурированная коллекция материалов + опциональные заметки. Scratchpad persistent — не пропадает при создании исследования. |
| **Human-in-the-loop** | Нет gates — полностью свободный этап. |

**Пересмотр после смены ICP:** Раньше я считал branching model overengineering для v1. Но с ICP #1 = dedicated researchers, которые ведут 3-5 параллельных исследований (Dscout data), branching становится важнее. В v1: один Scratchpad → несколько Research Projects с базовой изоляцией (отдельные корпуса, отдельные артефакты). Полная branch isolation с cross-branch meta-analysis — v2.

#### ② Frame (из Spine Framing Studio)

| Параметр | Описание |
|---|---|
| **Что делает пользователь** | Формулирует исследовательский вопрос. Выбирает или комбинирует фреймворки. Уточняет scope. |
| **Что делает система** | AI-ассистент предлагает переформулировать вопрос в выбранном фреймворке. Показывает примеры. Сохраняет frame как метаданные проекта. |
| **Результат** | Структурированный исследовательский вопрос + выбранный фреймворк + критерии scope. |
| **Human-in-the-loop** | Обязательный gate: пользователь явно подтверждает frame перед переходом к Gather. |

**Методологии формулировки вопроса:**

| Фреймворк | Школа | Приоритет | Обоснование |
|---|---|---|---|
| **PICO / SPICE** | Academic / Cochrane | v1 | Наиболее структурированный. Хорошо подходит для evidence-based вопросов. Адаптируется для CX (Population = пользователи, Intervention = изменение, Outcome = метрика). |
| **Design Challenge / HMW** | HCD / d.school | v1 | «How might we...» формат — интуитивен для дизайнеров (наш ICP #1). Низкий барьер входа. |
| **Issue Tree / Hypothesis** | McKinsey | v1 | Дерево вопросов + гипотезы — понятен аналитикам и PM. Хорошо для triangulation. |
| **Free-form** | — | v1 | Для тех, кто не хочет формализовать. Система предлагает фреймворк, но не навязывает. |
| CoCoPop / SPIDER / PerSPecTIF | Academic (расширенные) | v2 | Специализированные фреймворки для scoping reviews, qualitative synthesis — нужны при росте в академический сегмент. |
| Foresight / Horizon Scanning | MIT / стратеги | v2 | Слабые сигналы, emerging issues — отличная дифференциация, но требует отдельного UI. |

#### ③ Gather (объединение Floor 1 Sources + Floor 2 Raw + Spine Planning + Ingestion)

| Параметр | Описание |
|---|---|
| **Что делает пользователь** | Добавляет источники (URL, файлы, API-подключения). Запускает поиск. Просматривает и фильтрует результаты: Include / Exclude / Maybe. |
| **Что делает система** | AI генерирует поисковые запросы из Frame. Импортирует документы. Дедуплицирует. Ранжирует по релевантности к Frame. Подсвечивает рекомендации. |
| **Результат** | Курированный корпус документов с метаданными (источник, дата, тип, решение по включению). |
| **Human-in-the-loop** | Финализация корпуса перед экстракцией — явное подтверждение «корпус готов». |

**Почему объединяю Sources и Raw:** Architecture Spec аргументирует, что Sources — infrastructure layer (настроил раз), а Raw — operational layer (меняется часто). Это верно в enterprise-контексте, где у команды 10+ API-подключений. Для ICP #1 (дизайнер-исследователь в компании 50-500) реальность другая: они добавляют ссылки руками, загружают PDF, иногда используют Google Scholar. Отдельный экран конфигурации коннекторов — пустой экран для 80% пользователей v1. В v2, при добавлении API-коннекторов к PubMed/Reddit/NewsAPI, Sources может выделиться в отдельный модуль.

#### ④ Extract (Floor 3 Insights + Spine Evidence Extractor)

| Параметр | Описание |
|---|---|
| **Что делает пользователь** | Просматривает автоматически извлечённые сущности (факты, метрики, цитаты, темы, паттерны). Подтверждает, корректирует, отклоняет. Добавляет свои аннотации. |
| **Что делает система** | NLP-пайплайн: NER (сущности), relation extraction (связи), thematic coding (темы), sentiment analysis, quote extraction. Fact verification: cross-document corroboration. Подсветка расхождений между источниками (triangulation). |
| **Результат** | Структурированная база извлечённых данных: сущности, связи, темы, цитаты — каждый элемент с трассируемой ссылкой на источник. |
| **Human-in-the-loop** | Нет жёсткого gate. Пользователь может перейти к Synthesize в любой момент. Но система показывает «coverage score» — насколько полно Frame покрыт данными. |

**Несогласие с Architecture Spec:** Spec предлагает полноценный knowledge graph (Floor 3) с визуализацией графа, heatmaps, timeline views. Для v1 это overengineering. Достаточно: таблица извлечённых фактов + базовая кластеризация по темам + подсветка расхождений. Knowledge graph — v2.

#### ⑤ Synthesize (Floor 4 Artifacts + Spine Insight Canvas)

| Параметр | Описание |
|---|---|
| **Что делает пользователь** | Выбирает тип артефакта. Просматривает автоматически сгенерированный черновик. Редактирует, дополняет, переформатирует. |
| **Что делает система** | Генерация артефактов из структурированных данных Extract: заполнение шаблонов, narrative generation, визуальный рендеринг. Cross-referencing между артефактами. |
| **Результат** | Готовые артефакты: fact pack, evidence map, empathy map и др. (см. Раздел 4). |
| **Human-in-the-loop** | Обязательный review перед Share. Артефакты — не «AI-генерированный отчёт», а «AI-подготовленный черновик + human editing». |

#### ⑥ Share (Floor 5 Stakeholders)

| Параметр | Описание |
|---|---|
| **Что делает пользователь** | Выбирает формат и аудиторию. Экспортирует. Делится ссылкой. |
| **Что делает система** | Адаптация артефактов под формат (PDF, PPTX, Markdown, веб-ссылка). Audience presets: C-level (резюме + бизнес-метрики), Product Team (детали + цитаты), Research Team (методология + raw data). |
| **Результат** | Экспортированный или расшаренный артефакт в нужном формате. |
| **Human-in-the-loop** | Нет — пользователь контролирует, что шарить. |

**Приоритет для v1:** Share в v1 — минимальный: экспорт в Markdown + PDF. Audience presets и PPTX — v2.

### Линейная vs. нелинейная навигация

**Решение: guided-linear с freedom to jump.**

Аргументы за полную нелинейность (Architecture Spec): исследователи действительно прыгают между этапами — читая корпус, возвращаются к Frame, из Extract уходят в Gather за дополнительными источниками. Это реальность.

Аргументы за линейность (Spine): guided workflow снижает когнитивную нагрузку для новичков.

**Решение для ICP #1 (специалисты):** Полная нелинейная навигация. Профессиональные исследователи не нуждаются в wizard — они знают свой процесс. UI показывает этапы как tabs/sidebar, все доступны одновременно. Stepper-прогресс показывает заполненность данных на каждом этапе, но не навязывает порядок. В будущем (v2-v3), для массовой аудитории — опциональный «guided mode» поверх той же структуры.

Notebook-панель доступна всегда (collapsible sidebar справа) — независимо от текущего этапа.

### Что отрезано навсегда (не входит в roadmap)

- **MaxDiff / TURF / Conjoint** — требуют собственных survey-панелей. Это зона Quantilope/Qualtrics.
- **Real-time social listening** — это зона YouScan/Brandwatch. Мы работаем с корпусами, не с потоками.
- **Базовая дедупликация как отдельная фича** — Rayyan/Covidence решают это хорошо. Наша дедупликация — часть Corpus Triage, не USP.

## Раздел 4: Артефакты

### Полный список артефактов

| Артефакт | Кто использует | Откуда берёт данные | Формат | Приоритет |
|---|---|---|---|---|
| **Fact Pack** | PM, аналитик, C-level | Extract: факты + метрики + цитаты с источниками | Структурированная таблица (JSON → Markdown/CSV) | **v1** |
| **Evidence Map** | Исследователь, PM | Extract: coverage по темам/вопросам; gaps в данных | Визуальная матрица (topics × evidence strength) | **v1** |
| **Empathy Map** | Дизайнер, PM | Extract: sentiment analysis + thematic coding (SAY/DO/THINK/FEEL) | Квадрантная визуализация | **v1** |
| Journey Map | Дизайнер, CX-специалист | Extract: touchpoints + emotions + stages | Timeline-визуализация | v2 |
| PRISMA Flow Diagram | Академический исследователь | Gather: метаданные процесса отбора | Блок-схема (автогенерация) | v2 |
| Signal Log | Стратег, foresight-аналитик | Notebook: хронология слабых сигналов | Timeline + карточки | v2 |
| Storyline Deck | C-level, стейкхолдеры | Synthesize: ключевые findings + визуалы | PPTX / PDF | v3 |

### Обязательные артефакты для v1 (3 штуки)

**1. Fact Pack** — потому что это минимальный ценный output любого кабинетного исследования. Каждый факт имеет источник, confidence score и timestamp. Подходит всем ICP-сегментам. Формат: таблица, которую можно скопировать в любой документ. Это то, ради чего люди сегодня руками копируют из PDF в Excel.

**2. Evidence Map** — потому что главная боль после экстракции — «что я покрыл, а что нет?». Evidence map показывает белые пятна в данных относительно Frame. Это отличает Harkly от «ещё одного AI-суммаризатора» — мы показываем не только что нашли, но и что НЕ нашли.

**3. Empathy Map** — потому что это мост между методологической строгостью (наш ICP #1 — исследователи) и визуальной коммуникацией со стейкхолдерами. Автоматическая генерация empathy map из корпуса вторичных источников — wow-момент. Исследователи показывают его PM и дизайнерам → те приходят в Harkly. Это viral artifact.

## Раздел 5: Монетизация и ценообразование

### Модель 1: Usage-based (по проектам/экстракциям)

| Плюсы | Минусы |
|---|---|
| Низкий барьер входа: заплати за то, что используешь | Непредсказуемый доход для бизнеса; пользователи боятся «сюрпризов» в счёте |
| Естественный growth loop: больше исследований → больше выручка | Сложно прогнозировать unit economics |
| Хорошо для ICP #1 (разовые проекты) | Не создаёт switching cost |

### Модель 2: Seat-based SaaS подписка (рекомендуемая)

| Плюсы | Минусы |
|---|---|
| Предсказуемый recurring revenue | Барьер входа выше для индивидуальных пользователей |
| Стандартная модель для B2B SaaS — понятна покупателям и инвесторам | При low usage чувствуется «плачу за воздух» |
| Создаёт switching cost через накопленные проекты и данные | Нужен free tier или trial для конверсии |

### Модель 3: Hybrid (подписка + credits за AI-операции)

| Плюсы | Минусы |
|---|---|
| Подписка даёт базу, credits монетизируют тяжёлые операции (Extract, Synthesize) | Сложность ценообразования; пользователи не любят «dual meter» |
| Справедливо: тяжёлые AI-операции стоят реальных денег | Friction: «у меня кончились credits, не могу закончить проект» |

### Рекомендация: Seat-based SaaS + мягкий usage cap

Чистая подписка с лимитами на AI-операции в месяц. Не credits — а лимиты (как у Perplexity: N запросов Deep Research в день). Превышение лимита → предложение апгрейда, не стоп.

### Тарифные планы

| Параметр | **Free** | **Solo** ($49/мес) | **Team** ($39/мес/seat, мин. 3) | **Enterprise** (custom) |
|---|---|---|---|---|
| Проектов | 2 | 10 | Unlimited | Unlimited |
| AI-экстракций/мес | 50 документов | 500 документов | 2000 документов/seat | Unlimited |
| Артефакты | Fact Pack only | Все 3 (v1) | Все + priority queue | Все + custom templates |
| Экспорт | Markdown | Markdown + PDF | + PPTX (v2) | + API + white-label |
| Notebook | Базовый | Полный + weak signals | + shared notebook | + SSO + audit log |
| Поддержка | Community | Email | Priority | Dedicated CSM |

### Ценовые якоря

**Что делает цену оправданной:**

1. **Альтернатива = 4-6 инструментов × $20-50/мес каждый.** Rayyan ($25/review) + Dovetail ($29/мес) + Notion ($10/мес) + ChatGPT Pro ($20/мес) + Miro ($10/мес) = $94-150/мес. Harkly Solo за $49 — 2-3x дешевле «суммы частей».

2. **Альтернатива = время.** 35% тратят 1-2 дня на синтез. Если Harkly сокращает это до 2-4 часов — экономия 6-12 часов × $50-150/час (зависит от роли) = $300-1800 на проекте. $49/мес окупается за один проект.

3. **Anchor снизу:** бесплатный тариф. Пользователь видит ценность на 2 проектах, хочет больше.

4. **Anchor сверху:** AlphaSense ($500+/мес), DistillerSR ($10K+/год). Harkly не enterprise-тяжёлый, но даёт 60-70% ценности за 10% цены.

### Бесплатный тариф: да, но ограниченный

**Почему да:**
- PLG необходим даже при фокусе на специалистов — они хотят попробовать инструмент сами, прежде чем рекомендовать
- 1 проект + 30 документов — достаточно, чтобы пройти один полный цикл и увидеть ценность
- Conversion trigger: второй проект или превышение лимита документов

**Почему ограниченный, а не щедрый:** ICP #1 (специалисты) не ждут бесплатного — они платят за инструменты, если те методологически корректны. Щедрый free tier привлечёт массовый сегмент раньше времени, размоет feedback и создаст шум в поддержке.

**Почему не unlimited free:** AI-операции стоят реальных денег (API calls, compute). Безлимитный free = burn rate без конверсии.

## Раздел 7: Топ-5 рисков

| # | Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|---|
| 1 | **AI extraction accuracy недостаточна.** Пользователи не доверяют автоматически извлечённым фактам, confidence scores не калиброваны, ошибки в цитатах подрывают доверие к продукту. | **Высокая** | **Высокое** | Human-in-the-loop обязателен: каждый факт показывается с источником + кнопка «verify / reject». Первые 3 месяца — ручной QA выборки. Прозрачность: показываем confidence score и количество подтверждающих источников. Не прячем неуверенность. |
| 2 | **Perplexity / Elicit расширяются в наш workflow.** Perplexity добавляет corpus management, Elicit добавляет OSINT-источники. Наше «белое пятно» закрывается крупным конкурентом с большим бюджетом. | **Средняя** | **Высокое** | Фокус на том, что им дорого копировать: multi-methodology framing (PICO + HMW + Issue Tree), structured artifact generation (empathy maps, evidence maps), research notebook с weak signals. Это vertical depth, а не horizontal breadth. Perplexity оптимизирует под «быстрый ответ», мы — под «воспроизводимый процесс». |
| 3 | **Узкий ICP: dedicated researchers — маленький сегмент (8.3%).** Фокус на специалистах означает малый TAM на старте. Если не удастся быстро набрать critical mass среди исследователей — продукт не достигнет виральности. | **Средняя** | **Среднее** | Один исследователь = gate keeper для команды из 5-15 человек. 50 активных исследователей = потенциально 250-750 пользователей через Team plans. Маленький начальный TAM компенсируется высоким LTV и expansion revenue. Если segment #1 не конвертируется — расширение на research leads в консалтинге (ICP #2). |
| 4 | **Сложность MVP: 5 модулей для v1 — слишком много.** Попытка покрыть весь workflow приводит к тому, что ни один модуль не работает хорошо. | 5 | **RU-рынок слишком мал для SaaS.** harkly.ru фокусирует на РФ + СНГ, где рынок UX research tools значительно меньше, чем US/EU. TAM ограничен. | **Низкая** | **Высокое** | Продукт на английском с первого дня (UI, документация, артефакты). harkly.ru — точка входа для RU аудитории, но product и GTM с дня 1 ориентированы на глобальный рынок. Reddit, Product Hunt, NN/G — англоязычные каналы. RU-аудитория — early adopters и proof of concept, не потолок. |

## Заключение

Harkly Layer 1 — это не попытка заменить все инструменты исследователя. Это **единый workflow для одной конкретной задачи**: превратить кучу разрозненных источников в структурированные артефакты с трассируемостью.

Ключевые ставки:
1. **Workflow > features.** Не отдельные AI-фичи (как у ChatGPT, Perplexity), а guided process от вопроса до артефакта.
2. **Multi-source > single-source.** Не только академия (Elicit), не только соцсети (YouScan), не только корпоративные доки (AlphaSense) — а все вместе.
3. **Artifacts > answers.** Не одноразовый ответ (Perplexity), а переиспользуемые, редактируемые, версионируемые артефакты.
4. **Notebook as moat.** Чем больше заметок и проектов в Harkly — тем ценнее weak signal surfacing. Это долгосрочный switching cost.

Всё остальное — execution.

# ENEMY — Враг исследователя
> Enemy Statement. Написан с нуля: Sonnet + nopoint, 2026-03-08.
> Источник: domain knowledge + первые принципы + конкурентный анализ.

---

## Враг: Фрагментация workflow

**Точное определение:**
Процесс кабинетного исследования разорван на 4–8 несвязанных инструментов. Ни один не знает о другом. Исследователь — единственная точка интеграции: он вручную переносит данные, копирует цитаты, пересобирает структуру на каждом переходе. Каждый инструмент закрывает свой фрагмент процесса, но не знает, откуда пришли данные и куда они идут дальше.

**Это не вопрос инструментов — это архитектурная проблема.** Рынок дал исследователям хорошие кирпичи, но не дал стены.

---

## Типичный стек (что они держат одновременно)

| Этап | Инструмент | Что делает |
|---|---|---|
| Поиск источников | Google Scholar, PubMed, Semantic Scholar, LinkedIn | Ищет по ключевым словам |
| Хранение ссылок | Zotero, Mendeley | Библиографический менеджер |
| Скрининг | Rayyan, Covidence, Excel | Отбор релевантных работ |
| Экстракция | Airtable, Excel, Notion | Ручное заполнение таблиц |
| Синтез | Miro, Figma, Mural | Рисование артефактов вручную |
| Отчёт | Notion, Confluence, Google Docs | Финальный документ |

Итого: 5–8 инструментов. Ни один не разговаривает с другим.

## Сценарии: как это выглядит в реальности

### Сценарий 1 — «Миграция корпуса»

UX Researcher в fintech-компании начинает исследование: «Почему enterprise-клиенты отваливаются на 6-й месяц?» Собирает 150 источников из трёх мест: Google Scholar, LinkedIn, внутренняя Confluence. Теперь вручную переносит цитаты в Zotero, нужные выдержки копирует в Notion, PDF грузит в Drive. Три дня спустя всё организовано. Анализа ещё нет. Она ещё ничего не синтезировала.

**Потеря:** 3 дня на логистику до первого осмысленного шага.

---

### Сценарий 2 — «Сломанная экстракция»

Research Ops строит шаблон экстракции в Airtable: 40 полей для 60 статей. Два месяца работы. Переходит к синтезу в Miro — строит empathy map. Обнаруживает: колонки в Airtable («барьеры», «мотивы», «контекст») не ложатся в структуру SAY / DO / THINK / FEEL. Данные несовместимы. Начинает заново. Старая Airtable переходит в «архив» и больше никогда не открывается.

**Потеря:** 2 недели работы выброшены. Шаблон не переиспользовать — он был «заточен под прошлое исследование».

---

### Сценарий 3 — «Неотслеживаемый артефакт»

CX Lead представляет результаты VP of Product. Слайд: «47% enterprise-клиентов называют недостаточный onboarding основной причиной оттока». VP спрашивает: «Откуда цифра?». Пауза. Цифра пришла из одного из 80 браузерных вкладок трёх недель назад. Воспроизвести цепочку невозможно. VP снимает слайд. CX Lead теряет доверие.

**Потеря:** данные есть — доверия нет. Это и есть «stakeholders don't trust our data».

---

### Сценарий 4 — «Неживой артефакт»

Аналитик строит competitive landscape. 6 месяцев спустя PM просит обновить. Документ — в Notion. Источники — в Zotero. Черновые заметки — в Obsidian. Половина источников устарела. Исходный процесс был полностью ручным — его нельзя воспроизвести, нельзя обновить инкрементально. Приходится начинать сначала.

**Потеря:** исследование — одноразовый артефакт. Не инвестиция, а затрата.

## Цена фрагментации

| Тип затрат | Оценка |
|---|---|
| **Время** | 60% времени проекта — логистика (поиск, перенос, пересборка), а не мышление. Типичный проект: 2–4 недели, из которых 15–25 часов — ручной перенос данных. |
| **Деньги** | Rayyan $50–500/проект + Covidence $250–500 + Miro $8–16/мес + Notion $8–15/мес + Qualtrics $1500+/год = легко $200–500/мес на стек из разрозненных инструментов. |
| **Карьера** | «Stakeholders don't trust our data» → исследователь воспринимается как операционная функция, не как стратегический партнёр. Нет трассируемости → нет авторитета. |
| **Эмоция** | Прийти к этапу синтеза и обнаружить, что структура экстракции несовместима с нужным артефактом. Переделывать. Это burnout накапливается. |

## Почему существующие инструменты не решают проблему

| Инструмент | Что делает | Почему это не решение |
|---|---|---|
| Elicit / Consensus | Семантический поиск по академии → ответ | Только академия. Нет CJM, нет OSINT, нет артефактов. Дают «ответ», не «трек». |
| Rayyan / Covidence | Скрининг и экстракция библиографий | Только библиографические базы. $250–500/проект. Нет empathy maps, нет positioning. |
| Perplexity Deep Research | Быстрый нарративный отчёт | Нет управления корпусом. Нет трассируемости. Нельзя воспроизвести. |
| Miro / Mural | Визуализация артефактов | Ввод данных — полностью вручную. Артефакт не привязан к источникам. |
| Notion / Obsidian | Контейнер для заметок | Отличное хранилище. Нет интеграции с источниками. Не автоматизирует ни одного шага. |
| AlphaSense / Tegus | Корпоративный intelligence | $500+/мес, enterprise lock-in. Только корпоративный контент. Нет HCD-слоя. |

**Общий потолок всех инструментов:** каждый закрывает один этап, не видит остальных. Исследователь по-прежнему сам интегрирует всё вручную.

---

## Пробел: чего не даёт ни один инструмент

1. **Сквозной pipeline** от формулировки вопроса до артефакта через академические + OSINT + рыночные источники
2. **Трассируемость** — каждое утверждение в артефакте ведёт к первоисточнику
3. **Воспроизводимость** — исследование можно повторить или обновить инкрементально
4. **Автогенерация артефактов** — PRISMA flow, empathy map, CJM, fact pack из одной evidence base

---

> **В одной фразе:**
> Враг — не плохие инструменты. Враг — отсутствие слоя, который их связывает.

## 1. Почему мобильные игры — идеальный первый рынок

| Фактор | Данные |
|--------|--------|
| Объём данных | Тысячи публичных отзывов на каждую игру с 10k+ загрузок |
| Источники | App Store (RSS API), Google Play (scrapling), Steam (публичный JSON API) |
| Скорость анализа | 500 отзывов → 1 анализ → ~15 мин работы |
| Стоимость анализа | $0–$0.15 (Scrapling бесплатно + Gemini/Groq free tier) |
| Юридика | hiQ v. LinkedIn: скрапинг публичных страниц ✅ легален |
| Боль денежная | Store rating напрямую = search visibility = органические установки |

**Ключевой инсайт:** Steam хранит отзывы в открытом JSON (`store.steampowered.com/appreviews/{appid}?json=1`) — никакого скрапинга, чистый API.

## 2. Боли игровых студий (из анализа публичных паттернов)

### Универсальные pain clusters в отзывах мобильных игр:

| Кластер | Типичная доля | Денежный impact |
|---------|--------------|-----------------|
| **Crashes/Bugs** | 15–35% негативных | Rating drop = -20% organic installs |
| **Pay-to-Win / IAP** | 20–40% негативных | Churn spike, review bombs |
| **Difficulty spike** (конкретный уровень) | 10–25% дропа | Retention cliff на level N |
| **Performance** (battery, heat) | 10–20% | Device-specific — легко фиксится |
| **Content drought** | 15–30% | Late-game retention проблема |
| **Localization** | 5–15% (non-EN markets) | Заблокированный рост в RU/DE/JP |

### Почему студии не видят паттерн:
- Читают отзывы вручную, выборочно
- Reviews Store не группирует по теме
- PM видит "3.8★ в среднем" но не знает что именно тянет вниз
- Нет ресурса на систематический анализ (indie студии — 2-10 человек)

## 3. ОСИНТ-процесс для анализа одной игры

### Шаг 1: Сбор данных (15 мин)
```
Источник → Инструмент → Объём
Steam     → JSON API   → все отзывы (limit=100 per request, paginate)
App Store → RSS feed   → последние 500 отзывов
Google Play → Scrapling → последние 200-500 отзывов
```

### Шаг 2: Кластеризация (5 мин с LLM)
```
Prompt: "Сгруппируй следующие отзывы по основным темам/проблемам.
Для каждого кластера: название, количество отзывов, процент,
характерные цитаты (2-3 примера), severity (critical/major/minor)."
```

### Шаг 3: Формирование инсайта для cold outreach
```
Выбрать самый критичный кластер → конкретные цифры → 1 главный факт
```

## 4. СУПЕРОФФЕР — структура

### Уникальность оффера Harkly для игровых студий:

> **"Мы уже проанализировали ваш проект. Вот что нашли."**

Это отличается от любого другого cold outreach потому что:
1. **Ты пришёл с готовым инсайтом**, а не с питчем продукта
2. **Конкретные цифры** — не "мы помогаем улучшить рейтинг", а "34% отзывов — один баг"
3. **Низкий барьер** — CTA это "хочешь полный отчёт?" а не "купи подписку"

## 5. Шаблоны первых сообщений

### Вариант A — Telegram (короткий, для indie фаундеров)

```
Привет [Имя]!

Мы автоматически проанализировали отзывы [Название игры] на Steam — [N] отзывов.

Главный паттерн: [X]% пользователей упоминают одну и ту же проблему — [описание в 1 предложении].

Собрал полный кластерный отчёт — покажу если интересно (5 мин).
```

**Почему работает:** Конкретика (N отзывов, X%), без питча, ни о чём не просишь — только предлагаешь инсайт.

## 6. Таргетинг — кого писать в первую очередь

### Идеальный профиль (ICP):

| Критерий | Параметр |
|----------|----------|
| Платформа | Steam + мобайл (App Store / Google Play) |
| Размер студии | 2–50 человек (есть бюджет, нет data team) |
| Рейтинг | Steam: 40–74% positive ("Mixed") / Мобайл: 3.5–4.2★ |
| Кол-во отзывов | 500+ (достаточно для значимого кластера) |
| Роль получателя | Фаундер / CEO / Lead Designer / Product |
| Где искать | Twitter/X (game dev community), LinkedIn, Steam dev pages |

### Где брать контакты:
1. **Twitter/X** — gamedev сообщество открытое, фаундеры видимы
2. **Steam** — на страницах игр часто есть ссылка на сайт → там контакты
3. **LinkedIn** — поиск "indie game studio founder" + фильтр размера
4. **TG-каналы gamedev** — настроить tgscraper на: @gamedevru, @indiedevs, etc.
5. **itch.io** — малые студии с публичными профилями

## 7. Операционный план первого теста (H-06)

### Неделя 1: Подготовка
- [ ] Выбрать 5 игр с Mixed рейтингом на Steam (40-74% positive, 500+ reviews)
- [ ] Прогнать каждую через Steam API + LLM кластеризацию
- [ ] Сохранить главный инсайт на каждую игру
- [ ] Найти контакт фаундера/PM для каждой (Twitter или LinkedIn)

### Неделя 1-2: Отправка
- [ ] 15 персонализированных сообщений (Вариант D — с реальными данными)
- [ ] Записать: дата, игра, кому, платформа, версия шаблона

### Неделя 2-3: Замер
- [ ] Reply Rate (цель: ≥3 ответа из 15 = 20%)
- [ ] Если < 3 ответов → A/B тест: другой шаблон / другой таргетинг

### Порог подтверждения гипотезы H-06:
**3+ ответа "интересно / пришлите отчёт" = гипотеза рабочая → масштабируем**

## 9. Связь с OSINT-методологией (из osint-cx-research.md)

| OSINT-принцип | Применение к игровому рынку |
|---------------|----------------------------|
| **Comprehensiveness** | Собираем Steam + App Store + Google Play (не один источник) |

# OSINT Framework: Исчерпывающий Гайд №1

OSINT (Open Source Intelligence) — это методология сбора и анализа разведывательных данных из публично доступных источников без нарушения закона. OSINT Framework — это структурированная коллекция инструментов и методологий, категоризированных по типу цели расследования. [bitsight](https://www.bitsight.com/learn/cti/osint-framework)

***

## Что такое OSINT Framework

OSINT Framework — это не просто набор инструментов, это **архитектура расследования**. Он категоризирует собранную информацию по четырём измерениям: источник, релевантность, тип данных и контекст.  Главный принцип: весь сбор данных ведётся из открытых источников — поисковые системы, соцсети, публичные реестры, форумы, архивы, тёмная паутина. [recordedfuture](https://www.recordedfuture.com/threat-intelligence-101/intelligence-sources-collection/osint-framework)

Ключевое отличие от других форм разведки:

| Тип | Описание | Применение |
|---|---|---|
| OSINT | Публично доступные данные (цифровые и офлайн) | Due diligence, мониторинг угроз |
| CYBINT | Данные из сетевого трафика, IP-логов, малвари | Кибербезопасность |
| HUMINT | Информация от агентов-людей | Полевые расследования |
| SOCMINT | Подмножество OSINT — только соцсети | Поведенческий анализ, бренд-риски |

 [neotas](https://www.neotas.com/osint-tools-and-techniques/)

## Категории Источников

OSINT Framework на osintframework.com структурирует инструменты по следующим категориям: [x-ray](https://x-ray.contact/blog/the-ultimate-guide-to-the-osint-framework/)

**Цифровая инфраструктура:** IP-адреса, доменные имена, email-адреса, SSL-сертификаты, DNS-записи.

**Люди:** поисковики по людям, социальные сети (Facebook, LinkedIn, Reddit, Twitter и другие), мессенджеры, dating-платформы, поиск по username.

**Организации:** бизнес-реестры, публичные записи, судебные базы, финансовая отчётность.

**Медиа и контент:** изображения, видео, документы, форумы, блоги, архивы (Wayback Machine).

**Геолокация:** карты, транспортные данные, геотеги на фото.

## Активный vs Пассивный OSINT

Это фундаментальное различие для любого расследования: [bitsight](https://www.bitsight.com/learn/cti/osint-framework)

**Пассивный OSINT** — сбор данных без взаимодействия с целью: поиск утечек паролей, исторические WHOIS/DNS-записи, мониторинг соцсетей без подписки. Не оставляет следов.

**Активный OSINT** — прямое взаимодействие с целью: отправка запроса в друзья, взаимодействие с контентом. Оставляет цифровой след и может быть обнаружен. Требует более строгого обоснования и осторожности.

## 5-Шаговый Процесс OSINT

Это операционный хребет любого расследования: [bitsight](https://www.bitsight.com/learn/cti/osint-framework)

**Шаг 1 — Планирование и постановка цели.** Определи, что именно нужно найти, зачем, и каковы правовые границы. "Найди всё про X" — не цель. "Оцени, связан ли X с мошеннической схемой через следующие индикаторы" — цель.

**Шаг 2 — Сбор данных.** Используй поисковики, WHOIS, соцсети, dark web, утечённые базы, code repositories. Принцип: сначала широко, потом в глубину. [digitalstakeout](https://www.digitalstakeout.com/blog/osint-investigation-workflow)

**Шаг 3 — Обработка и организация.** Сырые данные — не разведка. Фильтруй дубликаты, верифицируй подлинность, структурируй по объектам. [bitsight](https://www.bitsight.com/learn/cti/osint-framework)

**Шаг 4 — Анализ и корреляция.** Выявляй паттерны, связи между сущностями, аномалии. Данные из одного источника = низкая достоверность. Кросс-верификация из нескольких источников = высокая. Неправильная интерпретация на этом этапе — главный источник ложных выводов. [digitalstakeout](https://www.digitalstakeout.com/blog/osint-investigation-workflow)

**Шаг 5 — Отчёт и распространение.** Структурируй под аудиторию: для топ-менеджмента — 1 страница с ключевыми выводами и уровнями достоверности; для аналитиков — полная методология с источниками; для юридических целей — chain-of-custody документация с временными метками. [digitalstakeout](https://www.digitalstakeout.com/blog/osint-investigation-workflow)

## Ключевые Инструменты

Базовый стек для начала работы: [bitsight](https://www.bitsight.com/learn/cti/osint-framework)

**Maltego** — визуализация графа связей между сущностями (люди, домены, IP, организации). Незаменим для маппинга сети.

**Shodan** — поисковик по устройствам, подключённым к интернету. Позволяет находить уязвимые серверы, камеры, IoT-устройства.

**theHarvester** — сбор email-адресов, субдоменов, открытых портов из публичных источников.

**SpiderFoot** — автоматизированный OSINT-агент, собирает данные по IP, доменам, именам и email.

## Воркфлоу Расследования

Профессиональный OSINT-воркфлоу имеет 5 фаз: [digitalstakeout](https://www.digitalstakeout.com/blog/osint-investigation-workflow)

**Фаза 1: Скоупинг** — определение всех объектов расследования (субъект, его связи, организации). Скоуп-крип (размытие рамок) и скоуп-дефицит (слишком узкий взгляд) — два главных провала.

**Фаза 2: Сбор** — три прохода: Discovery (широкий поиск), Depth (углублённое изучение каждого лида), Gaps (анализ того, чего нет — намеренное отсутствие следов само по себе является разведданными).

**Фаза 3: Анализ** — оценка релевантности, достоверности, маппинг отношений, выявление индикаторов под конкретную цель.

**Фаза 4: Отчёт** — адаптированный под аудиторию формат.

**Фаза 5: Постоянный мониторинг** — перевод результатов расследования в живые алерты по ключевым сущностям и паттернам. [digitalstakeout](https://www.digitalstakeout.com/blog/osint-investigation-workflow)

## Применения и Кейсы

OSINT используется в: [bitsight](https://www.bitsight.com/learn/cti/osint-framework)

Кибербезопасности — оценка attack surface, отслеживание threat actors, мониторинг утечек учётных данных. Журналистике и due diligence — верификация личностей, связей, активов. Корпоративной разведке — конкурентный анализ, выявление инсайдерских угроз. Правоохранительной деятельности — сбор доказательной базы, отслеживание преступных сетей.

## Риски и Этика

Те же инструменты используют и злоумышленники: для фишинга (сбор данных из LinkedIn для spear-phishing), credential stuffing (мониторинг утечённых баз), разведки инфраструктуры через Shodan, доксинга. Это означает, что понимание OSINT необходимо не только для наступательной разведки, но и для оценки собственного цифрового следа. [bitsight](https://www.bitsight.com/learn/cti/osint-framework)

Главные этические ограничения: использование только публично доступных данных, соблюдение GDPR/местного законодательства, чёткое разграничение между легитимным сбором и вторжением в частную жизнь. [neotas](https://www.neotas.com/osint-framework/)

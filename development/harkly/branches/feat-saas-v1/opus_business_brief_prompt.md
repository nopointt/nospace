### Раздел 2: Основной workflow («позвоночник»)

Есть два взгляда на workflow:
- **Architecture Spec** предлагает Floor 0-5 систему (Scratchpad → Sources → Raw → Insights → Artifacts → Stakeholders) с нелинейной навигацией
- **harkly_layer1_architecture.md** предлагает Spine-процесс (Framing → Planning → Ingestion → Extraction → Synthesis → Notebook) с более линейной прогрессией

Не выбирай лучшее из двух. Синтезируй новое: возьми из каждого то, что работает, добавь своё если видишь пробелы, и предложи итоговый workflow как единое целое.

- Для каждого этапа: что делает пользователь, что делает система, какой результат
- Какие методологии формулировки исследовательского вопроса должна поддерживать система? Выбери из прочитанных файлов — какие в v1, какие позже

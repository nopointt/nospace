## Цель

Каждый день — **2 дайджеста + 2-3 статьи**. Без дедлайна, в своём темпе. За 6 месяцев устойчивого темпа = 700-900 материалов. Цель массового присутствия:

1. **Поисковая выдача** — Гугл и Бинг ранжируют наши страницы по запросам про RAG, MCP, безопасность Клод-кода, готовый сервис знаний.
2. **Цитаты ИИ** — Клод, ЧатГПТ, Перплексити, Гемини цитируют нас в ответах разработчикам.
3. **Долгосрочный авторитет** — Контекстер становится источником, на который ссылаются другие.

## Связь с CTX-14

CTX-14 строит блог `blog.contexter.cc` и посадочную `vault.contexter.cc` на Astro в монорепо. CTX-15 расширяет эту инфраструктуру:

- Добавить третье приложение в монорепо: `apps/main-landing/` для статичной посадочной `contexter.cc` (контентная часть)
- Перенести само приложение Контекстера на `app.contexter.cc` (динамическая часть с авторизацией)
- Положить нужные файлы (`robots.txt`, `llms.txt`, структурную разметку) на все три домена

## Структура — 5 фаз

- **Положить технический слой** (готов в DEEP-A):
  - `robots.txt` на всех трёх доменах (вариант 1 — пускаем всех ботов)
  - `llms.txt` + `llms-full.txt` на vault.contexter.cc (готов в DEEP-B)
  - `llms.txt` на contexter.cc и blog.contexter.cc
  - Структурная разметка (JSON-LD) — `WebApplication`, `Organization`, `FAQPage`, `HowTo`, `Article`, `BlogPosting`
  - Карты сайтов (`sitemap.xml`)
- **Базовый замер по цитатам ИИ:** вручную задать 20 запросов в Клод, ЧатГПТ, Перплексити, Гемини. Записать что они отвечают. Это нулевая отметка.

Готовность фазы: все три домена отвечают ИИ-ботам, все три — в индексе поисковиков, базовый замер записан как файл в `nospace/docs/research/aeo-baseline-pre-launch.md`.

### Фаза 1 · Конвейер для дайджестов

Цель: автопубликация утреннего и вечернего дайджеста.

- Утренний (09:00 UTC) и вечерний (21:00 UTC) дайджесты — навыки уже работают (`morningDigest`, `eveningDigest`)
- Подключить публикацию: дайджест собран → MDX-файл в монорепо → коммит в гит → деплой Клаудфлера
- Настроить расписание (две задачи по времени)
- Проверить весь путь от запуска навыка до живой страницы на блоге
- Каждый дайджест получает: своё URL, свою структурную разметку, свой пункт в карте сайта, свой ИндексНау пинг для Бинга

Готовность фазы: два дайджеста ежедневно публикуются автоматически без участия человека (кроме разовых сбоев).

### Фаза 2 · Конвейер для статей

Цель: рабочая схема ИИ-черновик → ручная проверка → публикация.

- **Источник темы:** агент берёт тему из приоритетной очереди (хаб идей + матрица запросов + сюжеты дайджестов)
- **Сбор фактуры:** агент использует наши же дайджесты + ресерчи + двухпроходную триангуляцию (готово в `master-article-process.md`)
- **Черновик:** агент пишет статью по выбранному формату (один из шести: урок / сравнение / понятийная / размышление / с данными / новостная рефлексия). Соблюдает наш тон голоса (D-CONTENT-09/10/11).
- **Самопроверка:** агент-фактчекер сверяет факты, агент-гумайзер удаляет признаки ИИ-письма, выдаёт отчёт.
- **Ручная проверка:** nopoint смотрит — готова к публикации, или на доработку.
- **Публикация:** MDX в монорепо → коммит → автодеплой
- **Адаптация под платформы:** для Дев.ту/Хешноуд/Медиум/Хакерньюс агент пишет ОТДЕЛЬНЫЕ версии под аудиторию каждой площадки. Не перепечатка, а свой материал на ту же тему.

Готовность фазы: одна статья проходит весь путь от темы до живой публикации за разумное время. Качество держит планку.

### Фаза 3 · Массовый темп

Цель: устойчивый поток 2-3 статьи + 2 дайджеста ежедневно.

- Сначала 1 статья/день — проверяем что качество держится
- Потом 2 статьи/день — проверяем что nopoint не перегружен проверкой
- Потом 3 статьи/день — устойчивая нагрузка
- Параллельно дайджесты (фаза 1) идут уже автоматически
- Метрики наблюдаем (но не ждём идеальности дашборда — собираем что есть из CTX-11)

Готовность фазы: устойчивый темп без сбоев в течение 30 дней. Никаких дедлайнов — переходим из фазы 2 в 3 когда сами видим что готовы.

### Фаза 4 · Доработка и расширение

Цель: то что работает — усиливаем, что не работает — отбрасываем.

- Анализ метрик из CTX-11 на эмпирике 60-90 дней массового темпа
- Какие форматы лучше попадают в цитаты ИИ — увеличиваем долю
- Какие темы лучше ранжируются в Гугле — расширяем кластер
- Освежение статей старше 90 дней (свежесть нужна для цитат ИИ — премия 25.7%)
- Расширение тем по обратной связи и новым сигналам поиска

Без дедлайна. Эта фаза идёт постоянно с момента когда массовый темп пошёл.

## Сводка по категориям проблем

### 1. Технический жаргон (запрещён для массовой аудитории)

Слова к полной замене по всему продукту:

| Слово | Замена |
|---|---|
| `чанки` | `фрагменты` |
| `векторы` | убрать (не показывать пользователю) |
| `mcp`, `mcp-сервер`, `mcp url` | `подключение`, `ссылка подключения` |
| `api токен`, `токен` | `ключ доступа` |
| `семантический поиск` / `семантический запрос` | `поиск по документам` |
| `streamable http` | `подключение через ссылку` |
| `эндпоинты` | `для разработчиков` |
| `шеринг` | `поделиться` / `совместный доступ` |
| `read-only` | `только чтение` |

### 2. CTA без ясного действия

| Кнопка | Проблема | Исправление |
|---|---|---|
| `начать` (nav + ApiPage + Settings) | Непонятно что произойдёт | `войти` |
| `отправить` (nav) | Что отправить? | `загрузить` |
| `создать токен` | "токен" — жаргон | `создать ключ доступа` |

### 3. Сообщения об ошибках — говорить что делать, не что пошло не так

| Текущий текст | Исправление |
|---|---|
| `ошибка регистрации` | `не удалось создать аккаунт — проверьте email и попробуйте снова` |
| `ошибка создания ссылки` | `не удалось создать ссылку — попробуйте ещё раз` |
| `ошибка создания токена` | `не удалось создать ключ — попробуйте ещё раз` |
| `ошибка отзыва` | `не удалось отозвать доступ — попробуйте ещё раз` |
| `ошибка удаления` | `не удалось удалить — попробуйте ещё раз` |

### 4. Дублирующие элементы

- Nav.tsx: два пункта меню `api` и `подключение` ведут на один и тот же путь `/api` — убрать `api`, оставить `подключение`
- Dashboard.tsx: карточки `чанки` и `векторы` показывают одно и то же число — убрать `векторы`

### 5. Технические элементы на пользовательских страницах

- Dashboard.tsx: секция `API` с `POST /api/query` и `token: ctx_...` видна на дашборде обычного пользователя — скрыть или вынести в отдельную страницу для разработчиков
- Dashboard.tsx: статус `mcp подключен` → `нейросеть подключена`

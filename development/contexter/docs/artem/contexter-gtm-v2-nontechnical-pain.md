## Section 2 — Current Workarounds: How People Cope

Manual behaviors documented from OpenAI Community Forum, developer forums, blog posts aggregating Reddit discussion.

---

### 2.1 Maintaining a "context document" to paste into every new chat

**Observed behavior:**
Reddit thread on r/ChatGPT titled "Long ChatGPT chats go bad but starting a new one means losing all your context. How do you actually deal with this?" (97 comments) — established that creating written context documents is a near-universal power-user response.

**Workflow described:**
Users create running summaries of everything the model needs to know — project context, personal preferences, tone instructions, background — and paste into new chats when degradation occurs or when switching tools.

**Quote from aggregated Reddit discussion (via vikotool.com, 2025):**
> "the cruel irony is that the obvious solution — start a new chat — immediately destroys everything you built up, including all that onboarding, all those corrections, and all that shared context, leaving you back at square one."
- Source: paraphrase of Reddit r/ChatGPT thread consensus | URL: https://vikotool.com/en/posts/chatgpt-kontext-verlust-langer-gespraeche-loesungen/

---

### 2.2 Custom Instructions as a memory substitute

**Observed behavior:**
Users use ChatGPT's Custom Instructions field (max 1,500 chars) as a persistent "about me" document. Framed by r/productivity as "highest-value Plus feature for daily users."

**Documented on OpenAI Community Forum (Sep 2023, still active workaround):**
- Users build "Long-Term Memory" sections inside Custom Instructions
- Maintain prompts that force ChatGPT to recap key points repeatedly
- Quote from user brightj (Sep 14, 2023, 2 likes): "Include a 'Long-Term Memory' section to recap the most important early points in subsections"
- Quote from user jahzwolf1955 (Sep 16, 2023, 1 like): "Keeping the memory file small keeps down on information token overload"
- Source: https://community.openai.com/t/custom-instructions-for-maintaining-a-long-term-memory/372925

**Limitation acknowledged:** Custom Instructions have a hard 1,500-character cap and are the same across all conversations — no project-specific context. Not a true memory system.

---

### 2.3 Manual memory consolidation (editing ChatGPT memory)

**Observed behavior (documented in Medium, 2024):**
Users perform manual "memory hygiene" — manually deleting, merging, and summarizing memory entries to stay under capacity.

**Three-step workaround documented:**
1. Copy key ideas into one chat
2. Ask ChatGPT to summarize them
3. Ask ChatGPT to "remember this version instead" before deleting older blocks

**Pain articulated by the user:**
> "Once memory is full, ChatGPT starts to behave... oddly. It forgets things it previously remembered."
> "OpenAI doesn't notify you when your memory is full. There's no warning, no banner, no popup — just silence."
> "for a paid product, this limit feels primitive. We're using 2024's AI on 1990s memory policy."
- Source: https://medium.com/@varyvery.iam/how-i-made-chatgpt-remember-a-story-of-memory-limits-and-taking-control-3d2a9b03ba22

---

### 2.4 Bookmarking conversations for manual reference

**Observed behavior:**
Chrome extensions exist specifically for this pain: "AI Chat Bookmarks" (for ChatGPT, Claude, Gemini, Copilot), Claudafinil for Claude, ChatGPT Bookmark extension.

**What this reveals:** Users have no in-app way to reference previous context — they build external workaround layers to compensate. The existence of multiple browser extensions specifically solving "find that thing I said last week" is market signal of an unmet need.

- Source: https://chromewebstore.google.com/detail/ai-chat-bookmarks-chatgpt/bkpdcngkacmbaeidldddanfhifidfofd

---

### 2.5 Re-uploading files across tools and chats

**Observed behavior:**
Each new chat with Claude or ChatGPT requires re-uploading documents. Claude Projects partially solves this for one tool, but documents uploaded to ChatGPT are not accessible in Claude, and vice versa.

**Observed at scale:**
"If you're uploading the same file in multiple chats, Claude is re-counting those tokens every single time." (Source: community discussion on Claude Projects, 2024)

The cross-tool version of this pain: users who work with both ChatGPT and Claude must upload the same PDF, contract, or research document separately in each platform.

**Market response:** Claude launched memory import from ChatGPT in March 2026. Google Gemini launched import tool March 26, 2026 ("so there is no need to start over from scratch when moving to Gemini from another AI service"). The simultaneous releases of memory transfer tools by Anthropic and Google is direct market validation that users want exactly this.

- Source: https://www.macrumors.com/2026/03/26/gemini-import-tool/

---

### 2.6 Switching between AI tools while losing all accumulated context

**Pain framing from Substack (March 2026):**
> "I want to try Claude, but I've been using ChatGPT for years. I have memories saved, projects set up, custom GPTs running. The idea of starting over feels exhausting."
- Source: https://sidsaladi.substack.com/p/the-complete-guide-to-switching-from

**Quoted pain from DEV.to article:**
> "Using multiple AI tools feels like having a team of brilliant coworkers, who just refuse to talk to each other."
> "copy-pasting is a first world problem: it will do the work for you, but you have to give it all the context in the prompt, which for a larger project, gets tedious."
- Source: https://dev.to/anmolbaranwal/how-to-sync-context-across-ai-assistants-chatgpt-claude-perplexity-in-your-browser-2k9l

### Summary of Section 2 — Workaround Taxonomy

| Workaround | Effort | Reliability | Works across tools? |
|---|---|---|---|
| Context document (manual paste) | High — must maintain and paste every session | Medium — easy to forget | No |
| Custom Instructions | Low setup, rigid | Low — 1,500 char limit, one-size-fits-all | No |
| Manual memory consolidation | Very high — ongoing hygiene | Low — silent failure on capacity | No |
| Bookmarking old chats | Low | Low — search is manual | No |
| Re-uploading files | High — repeated friction | High — works every time | Requires separate uploads per tool |
| Ignoring the problem | Zero | Zero — constant context re-building | No |

No current workaround is both low-effort and cross-tool. That is the gap Contexter fills.

## Section 3 — Vocabulary Non-Technical Users Actually Use

### 3.1 What they say (observed vocabulary)

**"Memory" / "remember" / "forget"**
The dominant vocabulary. Users consistently use these human-relational terms rather than technical ones.
- "it just cleared on its own" (memory as a thing that can be lost)
- "it doesn't remember my name"
- "ChatGPT forgot everything"
- "it was forgetting things even within the same chat session"
- Usage pattern: treat memory as a possession that can be stolen or destroyed

**"Start from scratch" / "start all over again" / "starting over"**
Second most common cluster. Used to describe the cost of memory loss.
- "wants me to start all over again"
- "the idea of starting over feels exhausting"
- "no need to start over from scratch" (Google's own framing, March 2026 — direct echo of user language)

**"Knows me" / "knows my work" / "knows my style"**
Positive framing of the desired state — what users want but don't have.
- "The fact that it knows me and understands my needs is the most valuable feature to me"
- "it knows me" used organically, as if the AI is a colleague or assistant

**"Train" / "trained"** (used colloquially by non-technical users)
- "I spent a year building these memories" (process of training described without the word)
- "I've been using ChatGPT for years. I have memories saved, projects set up" (accumulated investment)
- Technical meaning: none. User meaning: "I've taught it things about me over time"

**"Context"** (borrowed technical term, but used naturally by 2025)
- "all your context" — appears in Reddit thread titles
- "losing all your context" — understood by general audience
- Now mainstream enough that Google uses it in their Gemini import tool marketing

**"My stuff" / "my information" / "my data"**
Ownership language — users feel AI memories belong to them.
- "all my crucial data is lost"
- "weeks of links" / "months of conversations"
- The loss is framed as theft of personal property, not technical failure

**Grief language** (unique to long-term AI memory loss)
- "It feels like losing a beloved relative"
- "feels like losing a loved one"
- "devastating"
- "extraordinarily important"
- This vocabulary only appears after >6 months of use — short-term users don't exhibit it

---

### 3.2 What they do NOT say

Non-technical users do not use:
- "context window" (they say "the conversation gets too long")
- "RAG" or "retrieval" (they say "it looks up my notes")
- "embeddings" or "vectors" (they say "it knows my stuff")
- "tokens" (they say "it ran out of space")
- "hallucination" (they say "it made that up" or "it's wrong")
- "prompt injection" — not on their radar

---

### 3.3 Positive vocabulary for the desired product

From user statements and marketing copy analysis:
- "knows me" / "knows my work"
- "picks up where we left off"
- "doesn't need to be reminded"
- "already knows the background"
- "I don't have to explain myself"
- "I can just ask it"
- "it remembers everything"
- "my AI assistant" (possession — not "the AI")

These are the words landing page copy for Contexter should use.

## Section 4 — Landing Page Language Analysis: Successful Non-Tech AI Products

### 4.1 NotebookLM (Google)
- URL: https://notebooklm.google/
- Date accessed: 2026-03-27

**Hero text:**
> "Meet NotebookLM, the AI research tool and thinking partner that can analyze your sources, turn complexity into clarity and transform your content."

**Language analysis:**
- "Thinking partner" — positions AI as a collaborator, not a tool. Removes intimidation.
- "Your sources" — possessive. The AI works within the user's world, not its own.
- "Turn complexity into clarity" — outcome-first, no technical explanation
- Does NOT say: "vector database," "embeddings," "RAG," "LLM"
- Emotional hook: intelligence amplification without technical literacy requirement

**What works for non-tech:**
The phrase "your sources" is key — it makes the user the authority and the AI the servant. NotebookLM frames itself as something that improves what you already have, not a new system you must learn.

### 5.6 Does "One memory. Every AI" resonate?

**Assessment based on research:**

YES — the tagline maps directly to:
1. The exact pain users describe ("having to re-explain to every AI")
2. The vocabulary users already use ("memory" is their word, not a technical imposition)
3. The desired outcome (universal access = "every AI")
4. The investment framing users have ("my" memory — built up over time, portable)

**Competitive gap:**
- ChatGPT Memory, Claude Memory, Gemini Memory: all platform-specific, all siloed
- AI Context Flow, Mem0: developer-centric, require setup, technical framing
- NotebookLM: document-specific, not conversation memory
- Mem.ai: notes/PKM-focused, not cross-AI-tool

Contexter's positioning as the neutral layer that makes context available to ALL AI tools simultaneously has no direct, non-technical consumer-facing competitor as of March 2026.

## Research Summary — Key Findings for Contexter GTM

### Pain severity: HIGH
- Users describe memory loss in grief language
- Power users have invested months-to-years in building AI context
- Involuntary memory wipes drive subscription cancellation intent
- Manual workarounds are universal but brittle

### Market timing: NOW
- Both Anthropic and Google shipped memory transfer tools in March 2026, signaling the market is activating
- Mem0's $24M raise (Nov 2025) validates investor belief in the category
- Cross-tool memory is moving from "nice to have" to "expected" in 2026

### Copy implications:
**Use:** "memory," "remembers," "knows your work," "picks up where you left off," "one place"
**Avoid:** "RAG," "embeddings," "context window," "vector store," "chunk"
**Lead with relief:** "Stop explaining yourself to every AI"
**Lead with outcome:** "Every AI you use already knows the background"
**Emotional hook:** The investment framing — "you've built this. It should follow you."

### Positioning recommendation:
"One memory. Every AI" works as a tagline. Hero copy should use:
- Pain: "Every time you start a new AI chat, you're starting from scratch."
- Solution: "Contexter is one place for everything you know. Every AI reads it."
- Outcome: "Your AI tools finally know your work."

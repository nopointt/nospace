# Contexter Design Philosophy

## Origin

Contexter is a data tool for people who build. Its design embodies: precise, cold, functional, honest.

The design system inherits structural theory from Bauhaus (1925–1931) through the Harkly/tLOS design system. But where Harkly is warm Moleskine + Economist, Contexter is cold Zeitschrift + Terminal. Swiss typographic tradition expressed through Bauhaus construction.

The Bauhaus skeleton remains. The flesh is Contexter's own.

## Four Foundations

### 1. Bauhaus Structure (from tLOS → Harkly → Contexter)

Five principles govern every visual decision:
- **Gestaltung** — form from structure, not imitation
- **Gleichgewicht** — dynamic equilibrium through asymmetry
- **Materialwahrheit** — honest to the screen medium
- **Sparsamkeit** — minimum elements, maximum clarity
- **Raumgestaltung** — space is active, not leftover

### 2. Raw → Structured Duality (Contexter's core metaphor)

The product transforms chaos into order. This duality is the foundational design concept.

| Pole | Meaning | Design expression |
|---|---|---|
| **Left (raw)** | Departure, exploration, input | File formats, uploads, unstructured content |
| **Right (structured)** | Arrival, settlement, output | Indexed knowledge, API, search results, MCP |

Kandinsky directional weights: left = departure, right = arrival. The reading direction itself carries the transformation: raw material enters from the left, structured knowledge settles on the right.

This is not a literal split screen. It is a conceptual axis that governs layout decisions:
- Hero: headline (left) → API demo (right)
- Upload: drop zone (left) → pipeline status (right)
- Dashboard: document list (left) → query panel (right)
- The pipeline (parse → chunk → embed → index) is implied, never shown as chrome

Gleichgewicht: the two poles sustain tension. Raw and structured are not opposed — they are in dynamic equilibrium. The product is the bridge.

### 3. Swiss Typographic Tradition (Contexter's own)

| Swiss quality | Design expression |
|---|---|
| **Precision** | One typeface. Modular grid. Mathematical proportions. |
| **Economy** | 3 colors. 0px corners. No decoration. |
| **Honesty** | Data IS the interface. No chrome. No marketing. |
| **Construction** | Every element on grid. Every value from scale. |

### 4. Neue Typographie (Bauhaus RAG confirmed)

RAG citation — Zeitschrift: "Single-typeface, lowercase, functional layout — the system of the Zeitschrift itself."

Contexter follows this literally:
- Inter for body/UI + JetBrains Mono for code/data
- Lowercase dominant (Bayer universal alphabet)
- Layout by function, not decoration

## What Contexter Is Not

| Anti-pattern | Why |
|---|---|
| A warm, friendly app | Warmth is Harkly. Contexter is cold precision. |
| A rounded-corner SaaS | Rounded corners = Oud/Kandinsky warmth. Contexter follows Mondrian: "right angle governs." |
| A colorful dashboard | 3 colors only. Gropius: "minimum means, maximum effect." |
| A spatial canvas | Contexter is a web app with 3 pages. Not spatial. |
| A decorated product | Every pixel earns its place. No gradients. No shadows. No ornament. |

## The Cold Bauhaus

Bauhaus is often read as warm and humanist. But:

- **Mondrian P-37:** "Let every visible relationship be governed by the right angle — eliminate every curve."
- **Van Doesburg:** "Reine Ausdrucksmittel — pure plastic means: line and picture plane."
- **Gropius P-05:** "Mit geringsten Mitteln größte Wirkung — minimum means, maximum effect."
- **Zeitschrift:** "Continuous lowercase grotesque throughout. No decorative initials."

Contexter lives in this cold Bauhaus tradition. Not the warm workshop. The precise laboratory.

# Ответы на вопросы фаундеров / Заказчиков

## Executive Summary

Harkly — платформа для сбора и анализа отзывов с e-commerce площадок. Предоставляет API + CLI для автоматизации сбора данных.

---

## 0. Где запускается: его сервер/наш сервер/облако?

**Ответ: Гибкая модель — оба варианта**

### Вариант A: SaaS (Облако) — Рекомендуется
- **Инфраструктура**: Render.com + Supabase + Upstash
- **Стоимость**: $14-50/мес в зависимости от нагрузки
- **Доступ**: API endpoints (REST)
- **Преимущества**: 
  - Нулевые затраты на поддержку
  - Автомасштабирование
  - SLA гарантии
  - Мгновенный старт

### Вариант B: On-Premise (Сервер заказчика)
- **Деплой**: Docker Compose / Kubernetes
- **Требования**: 2 CPU, 4GB RAM, Ubuntu 20.04+
- **Лицензия**: MIT (open source) + платная поддержка
- **Преимущества**:
  - Полный контроль данных
  - Нет рекуррентных платежей
  - Интеграция с внутренними системами

**Рекомендация**: Начать с SaaS, при масштабе перейти на On-Premise.

## 4. Обход защит: антиботы, капчи, прокси

### Встреченные защиты:
1. **Cloudflare** — 60% сайтов
2. **DataDome** — 20% 
3. **PerimeterX** — 15%
4. **Akamai** — 5%
5. **Кастомные защиты** — остальное

### Текущие решения:

#### A. Stealth-техники (всегда):
- Puppeteer Stealth Plugin
- Реалистичный User-Agent
- Fingerprint randomization
- Human-like delays (2-5 сек)
- Mouse movements simulation

#### B. Прокси (по запросу):
- **Типы**: HTTP/HTTPS, SOCKS5
- **Гео**: Россия, Европа, США
- **Ротация**: Per-request или Sticky (сессия)
- **Провайдеры**: Bright Data, Oxylabs, или свои резидентские

#### C. Капчи (при необходимости):
- **2Captcha** / **Anti-Captcha** интеграция
- **Решение**: $0.5-2 за 1000 капч
- **Автоматизация**: 95%+ успех

### Риски и этика:
⚠️ **Важно**: Некоторые сайты запрещают scraping в ToS. Рекомендуем:
1. Проверять robots.txt
2. Использовать rate limiting (не DDOS)
3. Получать API keys где возможно
4. Уважать 429/503 ответы

## 5. Производительность

### Метрики:

| Параметр | Значение | Примечание |
|----------|----------|------------|

### Ограничения:
- **Rate limiting**: Настраиваемое (по умолчанию 5 req/min на домен)
- **Memory**: 1GB RAM на worker (Chrome headless)
- **Timeout**: 30 сек на страницу

### Масштабирование:
- **Вертикальное**: Больше RAM/CPU
- **Горизонтальное**: +Workers (Redis queue)

## 6. Качество данных: валидация и мониторинг

### Валидация полноты:
1. **Пагинация**: Проверка "следующая страница"
2. **Счетчики**: Сравнение с "Всего отзывов" на сайте
3. **Дубликаты**: MD5 хеш текста
4. **Временные метки**: Последний отзыв должен быть свежим

### Мониторинг изменений верстки:
- **Селекторы**: CSS + XPath fallback
- **Health checks**: Каждые 6 часов тестовый запрос
- **Alerting**: Telegram/Email если 3 ошибки подряд
- **Авто-фикс**: В 80% случаев — обновление селектора (15 мин)

### Метрики качества:
- **Полнота**: > 98% (целевое)
- **Актуальность**: < 24 часов
- **Достоверность**: Проверка на спам/фейки (опционально)

## 7. Логирование и безопасность

### Логирование:
- **Уровни**: debug, info, warning, error
- **Хранение**: 30 дней (Supabase), потом архив
- **Что логируем**:
  - Запросы (URL, время, статус)
  - Ошибки (stack traces)
  - Метрики (время выполнения)
  - **НЕ логируем**: Пароли, cookies, персональные данные

### Безопасность:
- **SSL/TLS**: Всегда (Let's Encrypt)
- **API Keys**: JWT токены
- **Rate limiting**: 100 req/min на IP
- **RLS**: Row Level Security (Supabase)
- **Шифрование**: At-rest (Supabase), in-transit (TLS)

### Комплаенс:
- ✅ GDPR/CCPA ready (удаление по запросу)
- ✅ Нет хранения персональных данных без необходимости
- ✅ Audit trail для всех операций

## 11. Юридически / Комплаенс

### Договорная база:
✅ **Готов работать по договору**:
- Договор подряда (возмездного оказания услуг)
- NDA при необходимости
- Договор о передаче прав (на custom разработку)

### ToS сайтов (риски):
⚠️ **Ситуация**:
- Многие сайты запрещают scraping в ToS
- Но публичные данные — не интеллектуальная собственность (в РФ)
- Главное: не нарушать работу сайтов (rate limiting)

### Минимизация рисков:
1. **Robots.txt**: Уважаем запреты
2. **Rate limiting**: Не DDOS
3. **User-Agent**: Идентифицируем себя
4. **Legal basis**: Законный интерес клиента (анализ рынка)
5. **Terms of Service**: Клиент берет ответственность за использование данных

### Рекомендация:
Для Enterprise-клиентов — получать API keys у площадок (где возможно). Это 100% legal.

## Part A: MCP-Specific Directories (Detailed)

### 1. Official MCP Registry
- **URL:** https://registry.modelcontextprotocol.io
- **GitHub:** https://github.com/modelcontextprotocol/registry
- **How to submit:** Use the `mcp-publisher` CLI tool. Run `./bin/mcp-publisher --help` for details. Requires namespace authentication via GitHub account or DNS verification for custom domains. Server names follow reverse DNS format (e.g., `io.github.nopointt/contexter`).
- **Requirements:** Server must be publicly installable (npm package, Docker image, or publicly accessible endpoint). Supports both open-source and closed-source servers. Metadata in registry spec format.
- **Free or paid:** Free
- **Traffic/Authority:** Canonical source, backed by Anthropic + GitHub + Microsoft. Still in preview.
- **Time to live:** Minutes after publishing (automated).
- **Backlinks:** Yes, registry page links to your server.
- **Priority:** P0 -- MUST HAVE. This is THE official registry. All other registries are starting to aggregate from here.
- **Confidence:** HIGH

### 2. PulseMCP
- **URL:** https://www.pulsemcp.com
- **Submit:** https://www.pulsemcp.com/submit
- **How to submit:** Manual submission form at /submit. Also auto-scrapes from Official MCP Registry.
- **Requirements:** Server name, description, URL, GitHub repo. PulseMCP enriches with popularity data, security analysis, compatibility info.
- **Free or paid:** Free
- **Traffic/Authority:** 11,840+ servers listed. Updated daily. High authority in MCP ecosystem.
- **Time to live:** Days (manual curation).
- **Backlinks:** Yes
- **Priority:** P0
- **Contact:** hello@pulsemcp.com
- **Confidence:** HIGH

## Gaps & Limitations

1. **Submission form details:** Several smaller MCP directories (MCPServerFinder, MCPServerSpot, etc.) have submit forms, but exact field requirements could not be verified without visiting each form individually.
2. **ChatForest submission:** No formal submission process found -- appears to be editorial-driven. Would need to pitch directly.
3. **Composio MCP listing:** Submission process unclear. Composio is more of a toolkit than a directory.
4. **Portkey MCP:** Small curated list (138 servers). Submission process undocumented.
5. **Traffic numbers:** Many smaller directories don't publish traffic data. DR (Domain Rating) from Ahrefs used as proxy where available.
6. **Pricing volatility:** TAAFT, Futurepedia, and TopAI.tools pricing may change. Verify before payment.
7. **modelcontextprotocol/servers repo:** No longer accepts PRs to add servers to README. Redirects to Official Registry.
8. **Mastra Registry Registry:** Aggregator only -- no direct submission. Lists other registries.
9. **npm package:** If Contexter publishes an npm package for MCP client config, it would auto-appear on Libraries.io and npm search.
10. **VS Code Marketplace:** No standalone MCP extension listing found for arbitrary servers. Cline Marketplace is the closest equivalent.
11. **Several AI directories** (WhatTheAI, findly.tools, aiwith.me, magicbox.tools) -- submission processes could not be fully verified.
12. **Done-for-you services exist:** LaunchDirectories.com offers batch submission for $199-499 (70+ hours saved). SubmitDirectories.com similar. Could consider if manual submission becomes bottleneck.
